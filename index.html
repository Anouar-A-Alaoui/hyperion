About Hyperion

Hyperion is an advanced, modern directory tree visualizer written in Go. It provides a rich set of features for visualizing directory structures with extensive customization options.

Unlike traditional tree utilities, Hyperion offers colorized output, statistical analysis, file filtering, and visual charts showing file distribution - making it perfect for both day-to-day use and detailed filesystem analysis.

Usage

Hyperion offers a wide range of command-line options to customize its behavior. Here's the basic usage:

hyperion [flags]

Command Line Options

Flag Type Default Description
--path string "." Root directory to scan
--max-depth int -1 Maximum depth to recurse (-1 for unlimited)
Flag Type Default Description
--show-files bool false Whether to show files in output
--exclude-folders string[] ["node_modules"] Folders to exclude from tree (comma-separated)
--exclude-files string[] [] File extensions to exclude (comma-separated, e.g., '.exe,.dll')
--exclude-names string[] [] File names to exclude exactly (comma-separated, e.g., 'config.json,README.md')
Flag Type Default Description
--unicode bool true Use Unicode characters for pretty tree visuals
--color bool true Use colors in output
--bg-color bool false Use background color for items
--compact bool false Enable compact tree layout
Flag Type Default Description
--show-stats bool false Show total files, dirs, size
--stat-table bool false Show a table of largest files and types
--stats-count int 10 Number of top files to show in stats table
--chart bool false Show a visual chart of file size distribution

Documentation

Cross-Platform Support

Hyperion is designed to work seamlessly on both Linux and Windows systems. On Windows, it automatically detects Unicode support in the terminal and adjusts accordingly.

Tree Visualization Options

Hyperion provides three different chart renderers for visualizing file distribution:

  • Unicode Block Charts - Uses Unicode block characters for smooth chart visualization
  • ASCII Charts - Compatible with all terminals
  • Sparkline Charts - Compact representation using Unicode sparkline characters

File Filtering

Hyperion offers powerful filtering capabilities:

  • Exclude directories by name
  • Exclude files by extension
  • Exclude files by exact name
  • Limit traversal depth

Color Customization

When using colored output, Hyperion uses:

  • Blue for directories
  • Green for regular files
  • Magenta for symbolic links

With background color enabled, items are displayed with contrasting highlights for even better visibility.

Statistical Analysis

Hyperion provides comprehensive statistics about your filesystem:

  • Total directories and files
  • Total size in human-readable format
  • File type distribution
  • Largest files identification
  • Visual representation of file size distribution

Troubleshooting

Unicode Characters Not Displaying Correctly

If Unicode characters don't display correctly in your terminal, especially on Windows:

  • Use the --unicode=false flag to switch to ASCII mode
  • Make sure you're using a terminal that supports Unicode (like Windows Terminal instead of cmd.exe)
  • Verify your terminal font supports Unicode box drawing characters

Performance with Large Directories

For very large directories, you can improve performance by:

  • Using --exclude-folders to skip large folders like "node_modules", "vendor", etc.
  • Setting a reasonable depth limit with --max-depth
  • Excluding unnecessary files with --exclude-files
  • Using --compact mode for more efficient rendering

Terminal Compatibility

Hyperion is tested and works well with:

  • Linux: Terminal, GNOME Terminal, Konsole, xterm, and other standard terminals
  • Windows: Windows Terminal, PowerShell, Git Bash, WSL terminals

Integration with Scripts

Hyperion can be easily integrated into scripts or workflows:

# Save tree output to file
hyperion --show-files --unicode=false > tree.txt

# Generate statistics report
hyperion --show-stats --stat-table --chart > report.txt

# Analyze specific directory structure
hyperion --path="/path/to/project" --show-files --exclude-folders "node_modules,vendor" --chart

Analyzing Large Projects

For analyzing large software projects or codebases, try this approach:

hyperion --path="/path/to/project" --show-files --exclude-folders "node_modules,vendor,dist,build" --max-depth=4 --show-stats --stat-table --stats-count=15 --chart

This will:

  • Scan the project directory
  • Show files and directories
  • Exclude common large folders
  • Limit depth to 4 levels
  • Show statistics and file type distribution
  • Display a table of the 15 largest files
  • Include a visual chart of file size distribution
